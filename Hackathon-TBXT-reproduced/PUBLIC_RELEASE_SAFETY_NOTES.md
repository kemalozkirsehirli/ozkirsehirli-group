# Public Release Safety Notes

This file records changes made to reduce avoidable public-release risk. It is not legal advice and does not guarantee that no one can file a claim.

## Risk-reduction choices

1. **No personal allegations.** Public files avoid claims about personal conduct, discipline, removal, expulsion, or private interpersonal disputes.
2. **Neutral attribution.** Historical contributors are credited without assigning blame or publishing private narratives.
3. **No sole-ownership claim.** The repository does not assert that one person solely owns all preserved collaborative material.
4. **Maintainership boundary.** Current GitHub repository maintainership is separated from authorship and legal ownership.
5. **Third-party boundary.** Third-party software, datasets, models, APIs, compound catalogs, hosted bundles, organizer materials, and institutional names remain subject to their own terms.
6. **External asset IDs redacted.** Hardcoded public Google Drive IDs from setup/archive notes were replaced with placeholders or environment-variable instructions.
7. **Private/local paths neutralized.** Former local cluster paths and personal namespace paths were replaced with generic relative or placeholder paths where practical.
8. **No medical/procurement advice.** The repository states that computational predictions are not clinical, safety, procurement, synthesis, wet-lab, or regulatory instructions.
9. **Google Form privacy boundary.** The website and form template instruct users not to submit patient data, private medical information, passwords, confidential institutional data, or restricted third-party material.

## Remaining practical risk

A public repository can still attract copyright, contract, attribution, trademark, privacy, institutional, or defamation-related complaints. The safest public posture is factual, neutral, well-attributed, and clear about rights boundaries.
