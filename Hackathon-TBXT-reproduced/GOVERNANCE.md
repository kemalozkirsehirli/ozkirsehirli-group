# Governance

This file defines governance for the public `Hackathon-TBXT` GitHub repository.

## Current repository administration

Kemal Özkırşehirli / Özkırşehirli Group is the current public repository maintainer/contact.

The current maintainer has administrative responsibility for:

- public release tagging;
- pull-request review;
- documentation updates;
- issue moderation;
- attribution correction routing;
- repository security hygiene;
- license and permission questions to the extent of the maintainer's rights and knowledge.

## Governance boundaries

Repository governance does not adjudicate copyright ownership, patent ownership, inventorship, institutional rights, employment status, contract rights, or external disputes. Those questions may require separate written agreements or qualified legal review.

## Contributor process

Contributors may submit pull requests, issues, reproducibility reports, and attribution corrections. Contributions must not include confidential information, patient data, private compound catalogs, passwords, API keys, unpublished third-party datasets, or materials the contributor has no right to share.

## Attribution corrections

Attribution corrections should be handled neutrally and factually. Use the attribution-correction issue template and include evidence. Public issues should not be used for personal allegations or private disputes.

## Communication rule

Public communications about this repository should distinguish:

1. preserved historical research artifacts;
2. current public GitHub repository maintenance;
3. legal ownership or rights questions.

Do not collapse these into one claim.
