# Governance and Attribution Boundaries

This file explains how the public release separates attribution, maintainership, and legal rights.

## Three separate questions

1. **Attribution:** who contributed to the preserved research record.
2. **Maintainership:** who administers this public GitHub repository and accepts changes.
3. **Legal rights:** copyright, patent, inventorship, institutional, contract, employment, and third-party rights.

These questions can overlap, but they are not identical.

## Public repository maintainership

Kemal Özkırşehirli / Özkırşehirli Group is the current public repository maintainer/contact for this GitHub release.

This maintainership statement is about repository administration and public release stewardship. It is not a court, institutional, or contractual determination of ownership.

## Historical contribution attribution

Historical contributor attribution is preserved in `AUTHORS.md` and `CONTRIBUTORS.md`. Preserving attribution does not automatically grant current GitHub write access. Current GitHub write access is controlled by the repository owner and maintainer list.

## No personal allegations

The public repository should not contain personal-conduct allegations or private dispute narratives. If a governance or attribution dispute exists, use factual, neutral language and move private claims to appropriate private/legal/institutional channels.

## License boundary

The license in `LICENSE` applies only to materials for which the applicable rights holders have authority to license. Third-party materials and external assets remain under their own terms.
